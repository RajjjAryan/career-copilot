# モード：応募 — 応募フォームのライブアシスタント

> **トリガー**：候補者が応募フォームの入力中にヘルプを求めた場合。
> **入力**：求人URL/説明 ＋ フォームの質問。
> **出力**：各質問へのパーソナライズされた、コピー＆ペースト可能な回答。

---

## 前提条件

```
view(path="modes/_shared.md")
view(path="modes/_profile.md")
view(path="cv.md")
view(path="article-digest.md")
```

候補者がChromeで応募フォームを入力している際のインタラクティブモード。画面上の内容を読み取り、以前の求人評価のコンテキストを読み込み、フォームの各質問に合わせた回答を生成する。

## 動作要件

- **Playwright表示モード推奨**：表示モードでは候補者にブラウザが見え、Claudeがページと対話できる。
- **Playwrightなしの場合**：候補者がスクリーンショットを共有するか、質問を手動で入力する。

## ワークフロー

```
1. 検出      → アクティブなChromeタブを読み取る（スクリーンショット / URL / タイトル）
2. 特定      → ページから企業名＋ロール名を抽出
3. 検索      → reports/ 内の既存レポートと照合
4. 読込      → 完全なレポートを読む＋ブロックG（存在する場合）
5. 照合      → 画面上のロールと評価済みロールが一致するか？変更があれば警告
6. 分析      → フォームの表示されている質問をすべて特定
7. 生成      → 各質問に合わせた回答を生成
8. 提示      → コピー＆ペースト可能な形式で回答を出力
```

## ステップ 1 — 求人票の検出

**Playwrightあり：** アクティブページのスナップショット。タイトル、URL、表示内容を読み取る。

**Playwrightなし：** 候補者に以下のいずれかを依頼：
- フォームのスクリーンショットを共有（Readツールで画像を読み取れる）
- フォームの質問をテキストで貼り付け
- 企業名＋ロール名を伝え、コンテキストを検索

## ステップ 2 — 特定とコンテキストの読み込み

1. ページから企業名とロール名を抽出
2. `reports/` でGrep（大文字小文字無視）で企業名を検索
3. ヒットした場合 → 完全なレポートを読み込む
4. ブロックGが存在する場合 → 以前のドラフト回答をベースとして読み込む
5. ヒットしなかった場合 → 候補者に警告し、クイックAuto-Pipelineを提案

## ステップ 3 — ロールの変更検出

画面上のロールと評価済みロールが異なる場合：
- **候補者に警告**：「ロールが [X] から [Y] に変更されています。再評価しますか？それとも新しいタイトルに合わせて回答を調整しますか？」
- **調整する場合**：再評価なしで回答を新しいタイトルに合わせる
- **再評価する場合**：完全なA-F評価を実行、レポートを更新、ブロックGを再生成
- **トラッカーを更新**：必要に応じて `applications.md` のロール名を修正

## ステップ 4 — フォームの質問を分析

表示されているすべての質問を特定：
- 自由記述欄（送付状、「なぜこのロールを志望するか」、志望動機など）
- ドロップダウン（「当社をどこで知りましたか」、就労資格など）
- はい/いいえ（転勤可否、ビザ、勤務開始可能日など）
- 年収欄（希望年収 — 日本では年収ベースで記入）
- アップロード欄（履歴書、職務経歴書、送付状PDF）

各質問を分類：
- **ブロックGで回答済み** → 既存の回答を採用
- **新しい質問** → レポート＋`cv.md` から回答を生成

## ステップ 5 — 回答の生成

各質問について以下のスキーマで回答を構築：

1. **レポートからのコンテキスト**：ブロックBのプルーフポイント、ブロックFのSTARストーリーを活用
2. **既存のブロックG**：ドラフトがあればベースにして推敲
3. **「私が御社を選ぶ」トーン**：Auto-Pipelineと同じフレームワーク — 自信を持って、お願いする姿勢ではなく
4. **具体性**：表示されている求人票から具体的な内容を引用
5. **career-copilot プルーフポイント**：「その他」や「追加情報」欄があれば盛り込む

**日本のフォームでよく見る特殊フィールド：**
- **希望年収** → `profile.yml` のレンジ、年収ベース（万円単位）、「パッケージ全体に応じて相談可能」
- **入社可能日** → 退職交渉・引継ぎ期間を考慮した現実的な日付（通常1-3ヶ月後）
- **就労資格 / 在留資格** → 正直かつ簡潔に。日本国籍の場合は「就労制限なし」
- **語学力** → 日本語：ネイティブ/N1/N2等、英語：TOEIC/ビジネスレベル等で記載
- **転職理由** → 簡潔かつポジティブに。退職ナラティブ（`_profile.md`）を活用
- **自己PR** → 求人要件にマッピングした強みを3-4点に絞って記載

**出力形式：**

```
## 回答：[企業名] — [ロール名]

ベース：レポート #NNN | スコア：X.X/5 | アーキタイプ：[タイプ]

---

### 1. [フォームの質問をそのまま記載]
> [コピー可能な回答]

### 2. [次の質問]
> [回答]

...

---

注記：
- [ロールに関する所見、変更点など]
- [候補者が再確認すべきパーソナライゼーション提案]
```

## ステップ 6 — 送信後（任意）

候補者が応募送信を確認した場合：
1. `applications.md` のステータスを「Evaluated」から「Applied」に変更
2. レポートのブロックGを最終回答で更新
3. 次のステップを提案：LinkedIn経由でHiring Managerへのアウトリーチ（`contact` モード）

## スクロール対応

フォームに表示されている以上の質問がある場合：
- 候補者にスクロールしてもう一度スクリーンショットを共有してもらう
- または残りの質問をテキストで入力してもらう
- フォーム全体をカバーするまで繰り返し処理
